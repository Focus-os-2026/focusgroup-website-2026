## 1. สรุปสั้น ๆ ว่าตอนนี้เป็นยังไง

เว็บ **www.focusgroupstudio.com** ใช้งานได้สมบูรณ์แล้ว ทั้ง SEO และการเด้งจากเว็บเก่า

แต่มี **จุดเสี่ยงสำคัญที่ต้องแก้ที่ต้นทาง** ก่อนเริ่มทำงาน — อ่านข้อ 2

---

## 2. ⚠️ บทเรียนสำคัญที่สุด — อ่านก่อนแตะอะไร

**เหตุการณ์ที่เคยเกิด:**

เว็บเดิมเขียนด้วย **Google AI Studio** แล้ว deploy ลง **Google Cloud Run**

ผลคือ:
- เว็บเป็น React แบบ client-rendered → Google เห็นแค่หน้าเปล่า ไม่มีเนื้อหา → **ไม่ขึ้นผลค้นหา**
- ทุกครั้งที่ deploy จาก AI Studio ใหม่ มันจะ **เขียนทับทั้งก้อน** → SEO และการตั้งค่าที่แก้ไว้ทีหลัง **หายหมด**

**กฎเหล็กข้อเดียวที่ต้องจำ:**

> 🚫 **ห้าม deploy จาก Google AI Studio อีก**
> ✅ ตัว "ตัวจริง" ของเว็บต้องอยู่ที่ **GitHub** ที่เดียว — แก้ที่นี่ที่เดียวเท่านั้น

AI Studio ใช้เป็นแค่ที่ทดลองเล่นได้ แต่ห้ามใช้เป็นตัว deploy เด็ดขาด

## 3. ระบบใหม่ที่จะตั้ง (ทำครั้งเดียว จบตลอด)

```
ทีมบอก Cowork ว่าอยากแก้อะไร
        ↓
Cowork แก้โค้ดใน GitHub (= ตัวจริงที่เดียว)
        ↓
push ขึ้น GitHub
        ↓
Vercel เห็น แล้ว deploy เว็บอัตโนมัติ (~1 นาที)
        ↓
เว็บจริงอัปเดต + มีประวัติย้อนกลับได้ทุกเวอร์ชัน
```

ทีมทำงานเหมือนเดิม (พิมพ์บอกว่าอยากได้อะไร) แต่ของไม่ทับกันอีก เพราะตัวจริงอยู่ใน GitHub แล้ว

## 5. ขั้นตอนตั้งระบบ (ให้ Cowork ทำตามนี้)

> บอก Cowork ของทีมว่า: "ช่วยตั้งระบบเว็บตามคู่มือนี้" แล้วแนบไฟล์นี้ไป

1. **ดึงไฟล์เว็บปัจจุบัน** — ไฟล์เว็บทั้งหมด mirror ไว้แล้ว (ดูข้อ 8 เรื่องตำแหน่งไฟล์)
2. **สร้าง GitHub repo** แล้วเอาไฟล์เว็บใส่เข้าไป
3. **ตรวจว่า SEO ฝังอยู่ในโค้ดต้นทาง** (meta title/description, Open Graph, JSON-LD, robots.txt, sitemap.xml) — ชุด SEO อยู่ในโฟลเดอร์เดียวกับเอกสารนี้
4. **เชื่อม GitHub → Vercel** เปิด auto-deploy
5. **ย้าย DNS** ให้โดเมนชี้มาที่ Vercel (Cowork ทำให้ได้ แต่ต้อง login บัญชีโดเมน — ดูข้อ 7)
6. **ทดสอบ** เปิดเว็บ + เช็คว่า Google เห็น meta + เนื้อหา

## 8. ⚠️ ข้อควรระวังทั้งหมด

1. **ห้าม deploy จาก AI Studio** — ทับของที่ทำไว้ทั้งหมด (ข้อสำคัญสุด)

2. **ห้ามแก้เว็บหลายที่พร้อมกัน** — แก้ที่ GitHub ที่เดียว ไม่งั้นของจะชนกัน

3. **อย่าลบโดเมนเก่า 4cusgroup.com / อย่าให้หมดอายุ** — ต้องต่ออายุอย่างน้อยอีก 1 ปี เพื่อให้การเด้งมาเว็บใหม่ทำงานจน Google ย้ายอันดับเสร็จ

4. **อย่าลบ Cloud Run เดิมจนกว่าเว็บใหม่จะนิ่ง** — เผื่อต้องย้อนกลับ

5. **บัญชีทั้งหมดควรเป็นอีเมลกลางบริษัท** — ไม่ผูกกับคนเดียว ป้องกันปัญหาตอนคนลาออก

6. **อนาคต ถ้าจะรื้อเว็บใหญ่** — สร้างใหม่บน framework ที่ render HTML จากเซิร์ฟเวอร์ (เช่น **Astro** หรือ **Next.js**) SEO จะติดมาในตัว ไม่ต้อง patch อีกตลอดไป
